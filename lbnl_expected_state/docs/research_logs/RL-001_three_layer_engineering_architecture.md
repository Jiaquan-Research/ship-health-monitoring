# RL-001 — Three-Layer Engineering Architecture

## Purpose

Record a project-level conceptual observation.

This research log is exploratory.

It is not a frozen engineering decision.

## Summary

The project architecture naturally separates into three layers.

## Physics Layer

Expected State modeling.

This layer models expected physical behaviour from operating conditions.

## Engineering Layer

Transformation of statistical observations into Candidate Health Evidence.

This layer interprets residual behaviour through Statistical Validity, Semantic Admissibility, and controlled engineering experiments.

## Decision Layer

Translation of engineering evidence into maintenance decisions.

This layer requires operational stakeholder consensus before deployment.

## Interpretation

Different validation methodologies and different stakeholder consensus are required for each layer.

The Physics Layer can be validated statistically.

The Engineering Layer requires engineering interpretation and controlled experiments.

The Decision Layer requires operational stakeholder agreement.

## Status

Exploratory.

Not a frozen engineering decision.
