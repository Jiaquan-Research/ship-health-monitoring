# Publishing Policy

Document status: Governance policy
Task ID: MANIFEST_FIRST_REPOSITORY_GOVERNANCE_IMPLEMENTATION

## Purpose

This policy defines public visibility classes and publication rules for repository material. It does not authorize publication of internal, ambiguous-license, archive-only, or review-package-only material.

## Visibility Classes

`PUBLIC`: suitable for public use without special caveats.

`PUBLIC_WITH_BOUNDARIES`: suitable for public use only when the stated project boundaries are preserved.

`INTERNAL`: not intended for public publication without review.

`ARCHIVE_ONLY`: retained for historical traceability, not a current public source.

`REVIEW_PACKAGE_ONLY`: intended for controlled review package use, not general public browsing.

## Third-Party PDF and Licensing Review

Third-party PDFs, downloaded papers, and derived translations require licensing and citation review before public website publication. Their presence in the repository does not imply public redistribution approval.

## Personal Notes and Translations

Personal notes, Chinese translations, discussion drafts, and exploratory reading notes are not public authorities. They should be classified as `INTERNAL`, `ARCHIVE_ONLY`, or reviewed separately before publication.

## Prohibited Public Claims

Public pages must not state or imply:

- Marine validity is established.
- Marine feasibility is established.
- Deployment readiness is demonstrated.
- Shipboard RUL capability is validated.
- External Validation has started or completed.
- An External Reviewer has been selected.
- Marine data has been requested, received, or qualified.
- Deep Research review findings modify frozen claims or gaps.

## Website Pages as Derived Views

Website pages are derived public views. They must use manifest-selected sources and must not scan or publish the entire repository automatically. Website pages are never scientific or governance authorities.

## Publication Approval Workflow

1. Select source documents from the public manifest.
2. Confirm visibility class and boundary notes.
3. Confirm no prohibited public claims are introduced.
4. Confirm third-party material has licensing review if applicable.
5. Generate or update website content as a derived view.
6. Run link and state checks before publication.
